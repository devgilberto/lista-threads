🔹 Bibliotecas que usei
#include <stdio.h>

#include <pthread.h>
#include <unistd.h>
#include <string.h>

usei stdio pra ler e escrever, stdlib pra alocar memoria se precisar,
pthread pras threads, unistd pro sleep, e string pra manipular strings.

🔹 Constantes que defini
#define L 7
#define N_ARQS 5
#define T 3

L sao as 7 linhas da tela, N_ARQS eh a quantidade de arquivos que vao ser lidos,
T eh o numero de threads que o programa vai usar (tem que ser entre 1 e N_ARQS).

🔹 Mutexes que criei
pthread_mutex_t mutex_fila;
pthread_mutex_t mutex_tela[L];

um mutex pra controlar a fila de arquivos (quem pega qual arquivo),
e um array de 7 mutexes, um pra cada linha da tela. assim enquanto uma thread
muda a linha 3, outra pode mudar a linha 5 ao mesmo tempo. isso eh o tal do
fine-grained locking que o professor pediu.

🔹 Variavel global
int proximo_arquivo = 1;

essa variavel guarda qual eh o proximo arquivo a ser lido. comeco do 1 ate o 5.
as threads vao pegando os arquivos na ordem.

🔹 Pegar o proximo arquivo (secao critica)
pthread_mutex_lock(&mutex_fila);
if (proximo_arquivo > N_ARQS) {
    pthread_mutex_unlock(&mutex_fila);
    break;
}

pthread_mutex_unlock(&mutex_fila);

aqui eu travo o mutex da fila pra so uma thread por vez pegar um arquivo.
se ja acabaram os arquivos, a thread sai do loop (break).
senao, pega o id do arquivo e incrementa o contador.
